# 项目研究历程

## 阶段一：项目启动与基础搭建

**目标**：构建 PCB 二维尺寸自动测量的完整流程。

### 搭建内容
- 14 个源文件，模块化设计，参数集中在 `config.py`
- 命令行入口 `main.py`，支持 `calibrate | run_baseline | compare` 三个命令
- 输出目录结构：`outputs/calibration/`、`baseline/`、`light_corrected/`、`comparison/`

### 测量对象
- 同一块 PCB，标称尺寸 100×100 mm，安装孔中心距 94×94 mm
- 7 张不同光照条件下采集的图像

## 阶段三：基础测量算法开发（算法 A）

### 算法管线
```
原图 → 畸变校正 → HSV 绿色分割 → 形态学闭运算
    → 连通域候选筛选 → 四角点定位 → 透视矫正
    → 俯视图 ROI 孔检测 → 孔距/误差计算
```

### 候选筛选策略
- 面积占比 3%–25%
- 宽高比 0.75–1.33（接近方形）
- 不贴图像边界
- 中心不偏离
- 多边形逼近 + minAreaRect 备选
- 综合打分：方形度 (0.6) + 面积合理性 (0.4)

### 孔检测策略
- 俯视图中按理论位置开 ROI（15mm 半边长）
- ROI 内 CLAHE 增强 → 多源二值化 → 轮廓圆度筛选 → HoughCircles 备选

### 基础算法结果

| 图像 | 板检测 | 孔检测 | 孔距误差 |
|------|:------:|:------:|----------|
| 01 | ✅ | ✅ | 1.30 mm |
| 02 | ✅ | ✅ | 1.16 mm |
| 03 | ✅ | ✅ | 1.32 mm |
| 04 | ✅ | ✅ | 1.25 mm |
| 05 | ❌ | ❌ | — |
| 06 | ❌ | ❌ | — |

- **A 类板检测**：4/4 成功
- **A 类孔检测**：4/4 成功
- **A 类有效测量误差**：mean = 1.26 mm, std = 0.07 mm

### 孔检测策略

俯视图中按理论孔位开 ROI（15mm 半边长），ROI 内 CLAHE 增强 → 多源二值化 → 轮廓圆度筛选 → HoughCircles 备选。多候选评分公式：

```
score = 0.40 × circularity + 0.35 × area_score + 0.25 × position_score
```

其中 `position_score` 惩罚距离 ROI 理论中心过远的候选——ROI 本身就是用理论孔位开的，这个位置项不引入新先验，只是让同一 ROI 内的多个圆形候选优先选择靠近期望孔位的那一个。

### 失败原因分析
05、06 失败：桌面反光区域与 PCB 在 HSV 绿色 mask 中粘连为一个连通域，候选被贴边/面积过滤拦截。这是**诚实失败**，不是假阳性。

## 阶段四：光照矫正尝试（三次迭代，均未达预期）

### 初衷
通过光照矫正预处理，改善 05/06 的反光粘连问题，同时提升正常图像的测量精度。

### 第 1 版：V 通道全局除法校正（Retinex 路线）
- 大尺度高斯模糊估计光照场 → V / illumination
- **结果**：破坏颜色信号，平均误差 10.85 mm，比算法 A（1.26 mm）严重退化
- **放弃原因**：Retinex 除法改变了 HSV 各通道的相互关系，绿色分割失效

### 第 2 版：针对性高光抑制 ×0.8
- 只压制 V>190 & S<45 的反光区域，系数 0.8
- **结果**：压制力度不够，05/06 仍粘连，未恢复

### 第 3 版：强力压制 ×0.35 + 膨胀 mask
- 反光区 V 压制到光照场的 35%，膨胀 mask (15×15, 2 次迭代) 覆盖过渡带
- 增加 CLAHE 局部增强 + 暗区提升
- **结果**：
  - 05/06 仍未能恢复（压制仍不够强）
  - 已成功图像的精度**反而退化**（原本 1.16–1.32mm 退化到 1.7–6.6mm）
  - 实验组平均误差 5.89mm，比算法 A 的 1.26mm 严重退化

### 关键认识
**光照矫正陷入策略矛盾**：
- 要压断 05/06 的反光粘连需要很强的像素级处理
- 力度大了会破坏 02-04 的颜色信号，导致精度下降
- "全局改像素值"这条路在两类样本上效果互斥

## 阶段五：强光斑失效分析与研究方向转向

### 两类光照情况的识别

通过回顾阶段三的基础算法结果和阶段四的光照矫正实验结果，从光照干扰特征角度对样本进行分类：

| 类型 | 样本 | 特征 |
|------|------|------|
| A — 正常/轻反光 | 01, 02, 03, 04 | PCB 板面局部发亮，但不严重破坏区域分割 |
| B — 强光斑邻接 | 05, 06 | 强光斑紧贴 PCB 边缘，mask 连通域粘连 |

> 注：样本 07 经后续评估，特征不典型，从实验对象中移除。最终实验样本为 A 类 01-04、B 类 05-06，共 6 张。

### 核心反思

原研究框架"基础算法 vs 光照矫正算法谁更准"在实验上不成立——光照矫正在正常样本上精度退化，在强反光样本上未能恢复检测。真实情况是：

1. **A 类样本（01-04）**：基础算法已能完成端到端测量，问题在于精度而非可行性；光照矫正反而引入退化
2. **B 类样本（05-06）**：失效是 mask 层面连通域粘连导致的外框检测失败，不是像素值偏差问题
3. **改进方向**：应该在 mask 层面直接操作连通域拓扑，而非通过修改像素值间接修复

### 新研究定位

从"精度比较"转向"稳定性研究"：

> 基础算法在正常样本可工作；强光斑邻接条件下，区域分割发生粘连导致流程失败；改进算法通过高光感知与连通域约束，将 PCB 区域从反光干扰中分离；在保持正常样本精度基本不退化的前提下，提高强光斑样本的检测成功率和端到端测量稳定性。

### 新项目标题

**面向非均匀光照与局部强反光条件的 PCB 二维自动测量稳定性研究**

## 阶段六：架构重组

### 设计原则

1. 算法 A 与算法 B 的**后续测量链路完全一致**（外框定位、孔检测、孔距计算），避免"测量算法也变了"干扰实验结论
2. 算法 B **仅在 PCB mask 生成和区域分离环节**与算法 A 不同
3. 实验配置显式声明，通过 `ExperimentConfig` 控制策略切换

### 算法定义

**算法 A（基础测量算法）**：
```
畸变校正 → HSV PCB mask → 连通域候选筛选 → 四角点定位
         → 透视矫正 → 俯视图 ROI 孔检测 → 尺寸测量
```

**算法 B（强光斑感知改进算法）**：
```
畸变校正 → HSV PCB mask → 高光区域检测 → 光斑/PCB 粘连区域分离
         → 连通域候选筛选 → 后续测量流程同算法 A
```

算法 B 的关键约束：只改进 PCB mask 与区域分离，不改动后续测量链路。外框定位、孔位检测、孔距计算方法与算法 A 完全相同。

### 五层架构

```
src/
├── calibration/          # 层1: 相机标定（纯净解耦）
│   └── calibrate_camera.py
├── preprocessing/        # 层2: 可替换预处理策略
│   ├── baseline.py       #   普通预处理
│   ├── illumination.py   #   光照场估计
│   ├── highlight.py      #   高光检测与抑制
│   └── masks.py          #   mask 生成
├── board_detection/      # 层3: PCB 外框检测
│   ├── board_detector.py
│   ├── candidate_filter.py
│   ├── region_separation.py  # 【核心创新】
│   └── perspective.py
├── measurement/          # 层4: 几何测量
│   ├── hole_detector.py
│   └── geometry.py
└── experiments/          # 层5: 实验与报告
    ├── configs.py        #   ExperimentConfig
    ├── runner.py         #   统一执行引擎
    ├── statistics.py     #   分组统计
    └── report.py
```

### 关键设计决策

1. **预处理输出多种中间结果**（mask、光照场、高光 mask），检测模块按需选择，测量图像使用畸变校正原图
2. **`region_separation.py`**：在 mask 层面切除反光粘连，而非修改像素值——解决了阶段四的策略矛盾
3. **`ExperimentConfig`**：实验配置显式声明（preprocessing、board_mask、region_separation），避免代码分支膨胀
4. **样本分类纳入统计**：CSV 含 `sample_type` 字段，分组统计 A 类精度保持、B 类检测成功率

## 阶段七：核心创新 — region_separation.py

### 问题建模

B 类样本（05/06）的失效链路：

```
强光斑贴近 PCB 边缘
  → HSV 绿色 mask 中，光斑区域与 PCB 连通为单一连通域
  → 连通域贴边 → _is_near_boundary() 拦截
  → 外框检测失败
```

阶段四的光照矫正试图通过压制 V 通道切断这个链路，但像素级操作会连带影响 A 类样本。

### 方案：高光感知 mask 层面分离

直接在 mask 层面操作连通域拓扑，不修改像素值：

```
输入: green_mask, highlight_mask, image_shape

1. 连通域分析（connectedComponentsWithStats）
2. 对每个贴边界的连通域：
   a. 计算该连通域内高光 mask 的重叠率
   b. 重叠率 > 阈值（5%）→ 判定为光斑粘连
   c. 膨胀高光 mask（覆盖过渡带）
   d. 从 green_mask 中切除高光重叠区域
3. 形态学开运算断开残留细连接
4. 安全回退：分离后若无合法候选（candidate_filter 返回空）→ 返回原始 mask
```

### 与像素级光照矫正的对比

| 对比维度 | 像素级光照矫正（阶段四） | mask 层面分离（当前方案） |
|----------|---------------------------|---------------------------|
| 作用对象 | V 通道像素值 | 二值 mask 连通域 |
| 对 A 类样本的影响 | 必受影响（全局参数） | 自动跳过（贴边检查不触发） |
| 副作用 | 颜色信号退化、精度下降 | 仅改变连通域拓扑 |
| 回退机制 | 无（像素已修改） | 安全回退到原始 mask |
| 可解释性 | "V 值变了" | "切断了与光斑的连接" |

## 阶段九：四边形恢复模块 — quad_recovery.py（核心突破）

### 问题

阶段七实现的 `region_separation.py` 在 B 类样本上的实际表现：

```
05/06：高光分离触发 → 切除粘连区域 → 但分离后仍无合法候选 → 回退原始 mask → 失败
```

根本原因：B 类样本的光斑过大，即使通过高光 mask 引导切除，剩余 PCB 区域仍不能形成满足 `candidate_filter.py` 严格约束（面积 3%–25%、宽高比 0.75–1.33、不贴边、中心不偏离）的四边形候选。

**区域分离是必要步骤（切断了粘连），但它自身不足以产生合法候选。**

### 方案：逐行扫描四边形恢复

新建 `src/board_detection/quad_recovery.py`，作为 mask 层面候选筛选失败后的 fallback。

**核心思路**：放弃"从 mask 中找完整连通域→拟合四边形"的路线，改为"逐行扫描前景区间→找空间连续的稳定区域→拟合左右边界→构成四边形"。

**算法细节**：

```
输入: 二值 PCB mask（可能已被区域分离处理过，但仍不成形）

1. 形态学闭运算（9×9 矩形核，1次迭代）→ 弥合 mask 内部细小断裂

2. 逐行扫描：
   - 提取每行的前景区间 (foreground intervals)
   - 过滤：区间宽度 ≥ 8% 图宽，两端不贴图边界

3. 稳定区间序列分组 (_split_stable_runs)：
   - 相邻行、左右边界位置连续变化（容忍 20% 宽度跳变或 45px 偏移）
   - 形成多个稳定 run

4. 筛选：
   - run 高度 ≥ 20% 图高
   - 综合评分 = run 高度 × 平均宽度 × 宽度稳定性
   - 选最优 run

5. 双向线性拟合：
   - 去头尾 3% 行（避免边界毛刺）
   - 对左边界 x 值做 y→x 线性回归 → 左边界线
   - 对右边界 x 值做 y→x 线性回归 → 右边界线
   - 取 run 的 y_min、y_max → 四角点

6. 验证：
   - 角点在图像范围内
   - 四边形面积 ≥ 1% 图面积
   - 填充率 (fill_ratio) = mask 前景面积 / 四边形面积

7. 输出: (corners_4x2, score, diagnostics)
```

**关键设计约束**：

此模块**仅使用一条目标先验**："目标外框可以表示为四边形"。不使用：
- ❌ 孔距 (94mm)
- ❌ 板尺寸 (100×100mm)
- ❌ 正方形假设（宽高比≈1.0）
- ❌ 固定位置（PCB 在图像中央）

这使得恢复模块的结论具有独立性——四边形恢复成功不是"因为知道了答案"，而是"mask 证据确实支持存在一个四边形区域"。

### 算法 B fallback 链

修改 `runner.py`，算法 B 的板检测流程变为：

```
segment_pcb_green (HSV 绿色分割)
    → separate_highlight_from_mask (区域分离)
    → find_board_contours (常规候选筛选)
        ✅ 成功 → 正常测量
        ❌ 失败 → recover_quad_from_mask (行扫描恢复)
                    ✅ 成功 → 测量（quad_recovery_used=True）
                    ❌ 失败 → 报 FAIL
```

**关键设计**：`detect_board()` 已有 `candidates_override` 参数，可以直接注入恢复出的四边形作为候选。四边形恢复后再走完全相同的透视矫正、孔检测、测量流程——**不改动后续测量链路**。

## 阶段十：测量图像分离（精度修复）

### 问题

阶段四遗留问题：算法 B 在 A 类样本上的精度退化（+3.88mm）。根因确认：

```
preprocess_highlight_aware() 的输出:
  高光抑制 → CLAHE → 灰度化 → ...
  这张"矫正图"同时用于:
    (a) 生成 mask / 找四边形  ← 需要高光抑制
    (b) 透视矫正 + 孔检测     ← 不应做高光抑制！
```

高光抑制改变了 V 通道的亮度分布 → 灰度图结构发生变化 → 外框定位和孔检测精度受影响。

### 方案

修改 `preprocessing/__init__.py` 的 `preprocess_highlight_aware()`：

```python
# 新增字段：只做畸变校正，不做任何光照处理
results["undistorted"] = corrected_bgr   # 畸变校正原图

# ... 后续高光抑制、CLAHE 只作用于 corrected
results["corrected"] = corrected_final_bgr  # 高光处理图
```

修改 `runner.py`：

```python
# 高光处理图 → 用于 mask 和四边形检测
green_mask = segment_pcb_green(corrected)

# 畸变校正原图 → 用于测量
measurement_image = preproc_results.get("undistorted", corrected)
board_result = detect_board(measurement_image, mask_override=green_mask)
```

**职责分离**：
- `corrected`：mask 生成、区域分离、四边形恢复——可以用激进的像素处理，因为不参与测量
- `undistorted`：透视矫正、孔检测、尺寸计算——保持原始灰度结构，精度不受影响

### 效果

对 05 的提升最为显著：孔距误差从无法测量 → **0.66 mm**。

## 阶段十一：算法 B 运行范围限定

### 决策

算法 B 不再在 A 类样本上运行。

**理由**：

1. **科研逻辑**：算法 B 的设计目标是解决 B 类强光斑邻接问题。在 A 类上跑算法 B 回答不了任何研究问题
2. **统计清洁**：阶段八设计的"第三类对比（A vs B 在 A 类上）"本质上是验证"不退化"，但这不是研究贡献——即使在 A 类上退化，只要 B 类恢复效果好仍然有价值
3. **报告聚焦**：删除 A 类算法 B 对比，让报告的两条主线更清晰：
   - 算法 A 在 A/B 上的表现 → 显示常规流程的失效边界
   - 算法 B 在 B 上的表现 → 显示改进是否有效

### 修改

- `main.py`：新增 `_filter_b_samples()`，`algorithm-b` 和 `compare` 中算法 B 仅处理 B 类
- `statistics.py`：删除第三节（A 类辅助对比），新增四边形恢复统计小节
- `runner.py`：`print_grouped_summary()` 对算法 B 仅展示 B 类统计

### 最终实验对比结构

```
1. 算法 A 在 A/B 两类样本上的运行结果
   — A 类：4/4 成功，误差 mean=1.26mm ± 0.07mm
   — B 类：0/2 成功 → 常规流程在强光斑邻接下失效

2. 算法 A vs 算法 B 在 B 类样本上的对比  ← 唯一核心对比
   — 算法 A：0/2 板检测，0/2 孔检测
   — 算法 B：2/2 板检测，2/2 孔检测
   — 四边形恢复触发：2/2
   — 分离效果：连通域 1223→73.7（削减 94%）
```

### 核心科研结论

1. **算法 A 的基础性能**：在正常/轻反光条件下，基础测量流程可靠（A 类 4/4 成功），测量精度 1.16–1.32 mm，重复性 std = 0.07 mm
2. **强光斑的失效机制**：光斑紧贴 PCB 边缘 → HSV 绿色 mask 中 PCB 与光斑连通域粘连 → 候选被贴边/面积/宽高比约束拦截 → 外框检测失败
3. **像素级光照矫正的局限**（经三个阶段迭代确认）：在"压制强反光"和"保持正常图像精度"之间存在不可调和的策略矛盾
4. **mask 层面分离 + 四边形恢复的有效性**：区域分离切断粘连 + row-interval 四边形恢复从残损 mask 中重建外框，成功将 B 类样本从"完全无法测量"恢复为"端到端可测量"（2/2 成功，误差均 < 1 mm）
5. **测量图像分离的必要性**：将高光处理图仅用于 mask 生成、原始畸变校正图用于测量，避免了精度退化
6. **孔检测位置先验**：在多候选评分中加入 ROI 中心距离项，利用理论孔位开 ROI 的既有约束，不引入新先验，显著降低误选焊盘/小孔的风险
