# 项目研究历程

## 阶段一：项目启动与基础搭建

**目标**：构建 PCB 二维尺寸自动测量的完整流程。

### 搭建内容
- 14 个源文件，模块化设计，参数集中在 `config.py`
- 命令行入口 `main.py`，支持 `calibrate | run_baseline | compare` 三个命令
- 输出目录结构：`outputs/calibration/`、`baseline/`、`light_corrected/`、`comparison/`

### 测量对象
- 同一块 PCB，标称尺寸 100×100 mm，安装孔中心距 94×94 mm
- 7 张不同光照条件下采集的图像

## 阶段二：相机标定

- 棋盘格内角点 (10, 10)，单格 10mm，A4 纸打印

## 阶段三：基础测量算法开发（算法 A）

### 算法管线
```
原图 → 畸变校正 → HSV 绿色分割 → 形态学闭运算
    → 连通域候选筛选 → 四角点定位 → 透视矫正
    → 俯视图 ROI 孔检测 → 孔距/误差计算
```

### 候选筛选策略
- 面积占比 3%–25%
- 宽高比 0.75–1.33（接近方形）
- 不贴图像边界
- 中心不偏离
- 多边形逼近 + minAreaRect 备选
- 综合打分：方形度 (0.6) + 面积合理性 (0.4)

### 孔检测策略
- 俯视图中按理论位置开 ROI（15mm 半边长）
- ROI 内 CLAHE 增强 → 多源二值化 → 轮廓圆度筛选 → HoughCircles 备选

### 基础算法结果

| 图像 | 板检测 | X 孔距 | 误差 |
|------|--------|--------|------|
| 01 | ✅ | 86.86 mm | 6.29 mm |
| 02 | ✅ | 92.06 mm | 1.79 mm |
| 03 | ✅ | 95.38 mm | **1.32 mm** |
| 04 | ✅ | 95.09 mm | **1.25 mm** |
| 05 | ❌ | — | — |
| 06 | ❌ | — | — |
| 07 | ✅ | 92.07 mm | 2.89 mm |

- **板检测**：5/7 成功
- **孔检测**：检测到的板子 5/5 全部成功
- **有效测量误差**：1.25–6.29 mm（均值 2.71 mm）

### 失败原因分析
05、06 失败：桌面反光区域与 PCB 在 HSV 绿色 mask 中粘连为一个连通域，候选被贴边/面积过滤拦截。这是**诚实失败**，不是假阳性。

## 阶段四：光照矫正尝试（三次迭代，均未达预期）

### 初衷
通过光照矫正预处理，改善 05/06 的反光粘连问题，同时提升正常图像的测量精度。

### 第 1 版：V 通道全局除法校正（Retinex 路线）
- 大尺度高斯模糊估计光照场 → V / illumination
- **结果**：破坏颜色信号，平均误差 10.85 mm，比 baseline 更差
- **放弃原因**：Retinex 除法改变了 HSV 各通道的相互关系，绿色分割失效

### 第 2 版：针对性高光抑制 ×0.8
- 只压制 V>190 & S<45 的反光区域，系数 0.8
- **结果**：压制力度不够，05/06 仍粘连，未恢复

### 第 3 版：强力压制 ×0.35 + 膨胀 mask
- 反光区 V 压制到光照场的 35%，膨胀 mask (15×15, 2 次迭代) 覆盖过渡带
- 增加 CLAHE 局部增强 + 暗区提升
- **结果**：
  - 05/06 仍未能恢复（压制仍不够强）
  - 02-04 已成功图像的精度**反而退化**（1.2mm → 1.7~6.6mm）
  - 实验组平均误差 5.89mm，比 baseline 2.71mm 更差

### 关键认识
**光照矫正陷入策略矛盾**：
- 要压断 05/06 的反光粘连需要很强的像素级处理
- 力度大了会破坏 02-04 的颜色信号，导致精度下降
- "全局改像素值"这条路在两类样本上效果互斥

## 阶段五：强光斑失效分析与研究方向转向

### 两类光照情况的识别

通过回顾阶段三的基础算法结果和阶段四的光照矫正实验结果，从光照干扰特征角度对样本进行分类：

| 类型 | 样本 | 特征 |
|------|------|------|
| A — 正常/轻反光 | 01, 02, 03, 04 | PCB 板面局部发亮，但不严重破坏区域分割 |
| B — 强光斑邻接 | 05, 06 | 强光斑紧贴 PCB 边缘，mask 连通域粘连 |

> 注：样本 07 经后续评估，特征不典型，从实验对象中移除。最终实验样本为 A 类 01-04、B 类 05-06，共 6 张。

### 核心反思

原研究框架"基础算法 vs 光照矫正算法谁更准"在实验上不成立——光照矫正在正常样本上精度退化，在强反光样本上未能恢复检测。真实情况是：

1. **A 类样本（01-04）**：基础算法已能完成端到端测量，问题在于精度而非可行性；光照矫正反而引入退化
2. **B 类样本（05-06）**：失效是 mask 层面连通域粘连导致的外框检测失败，不是像素值偏差问题
3. **改进方向**：应该在 mask 层面直接操作连通域拓扑，而非通过修改像素值间接修复

### 新研究定位

从"精度比较"转向"稳定性研究"：

> 基础算法在正常样本可工作；强光斑邻接条件下，区域分割发生粘连导致流程失败；改进算法通过高光感知与连通域约束，将 PCB 区域从反光干扰中分离；在保持正常样本精度基本不退化的前提下，提高强光斑样本的检测成功率和端到端测量稳定性。

### 新项目标题

**面向非均匀光照与局部强反光条件的 PCB 二维自动测量稳定性研究**

## 阶段六：架构重组

### 设计原则

1. 算法 A 与算法 B 的**后续测量链路完全一致**（外框定位、孔检测、孔距计算），避免"测量算法也变了"干扰实验结论
2. 算法 B **仅在 PCB mask 生成和区域分离环节**与算法 A 不同
3. 实验配置显式声明，通过 `ExperimentConfig` 控制策略切换

### 算法定义

**算法 A（基础测量算法）**：
```
畸变校正 → HSV PCB mask → 连通域候选筛选 → 四角点定位
         → 透视矫正 → 俯视图 ROI 孔检测 → 尺寸测量
```

**算法 B（强光斑感知改进算法）**：
```
畸变校正 → HSV PCB mask → 高光区域检测 → 光斑/PCB 粘连区域分离
         → 连通域候选筛选 → 后续测量流程同算法 A
```

算法 B 的关键约束：只改进 PCB mask 与区域分离，不改动后续测量链路。外框定位、孔位检测、孔距计算方法与算法 A 完全相同。

### 五层架构

```
src/
├── calibration/          # 层1: 相机标定（纯净解耦）
│   └── calibrate_camera.py
├── preprocessing/        # 层2: 可替换预处理策略
│   ├── baseline.py       #   普通预处理
│   ├── illumination.py   #   光照场估计
│   ├── highlight.py      #   高光检测与抑制
│   └── masks.py          #   mask 生成
├── board_detection/      # 层3: PCB 外框检测
│   ├── board_detector.py
│   ├── candidate_filter.py
│   ├── region_separation.py  # 【核心创新】
│   └── perspective.py
├── measurement/          # 层4: 几何测量
│   ├── hole_detector.py
│   └── geometry.py
└── experiments/          # 层5: 实验与报告
    ├── configs.py        #   ExperimentConfig
    ├── runner.py         #   统一执行引擎
    ├── statistics.py     #   分组统计
    └── report.py
```

### 关键设计决策

1. **预处理输出多种中间结果**（mask、光照场、高光 mask），检测模块按需选择，测量图像使用畸变校正原图
2. **`region_separation.py`**：在 mask 层面切除反光粘连，而非修改像素值——解决了阶段四的策略矛盾
3. **`ExperimentConfig`**：实验配置显式声明（preprocessing、board_mask、region_separation），避免代码分支膨胀
4. **样本分类纳入统计**：CSV 含 `sample_type` 字段，分组统计 A 类精度保持、B 类检测成功率

## 阶段七：核心创新 — region_separation.py

### 问题建模

B 类样本（05/06）的失效链路：

```
强光斑贴近 PCB 边缘
  → HSV 绿色 mask 中，光斑区域与 PCB 连通为单一连通域
  → 连通域贴边 → _is_near_boundary() 拦截
  → 外框检测失败
```

阶段四的光照矫正试图通过压制 V 通道切断这个链路，但像素级操作会连带影响 A 类样本。

### 方案：高光感知 mask 层面分离

直接在 mask 层面操作连通域拓扑，不修改像素值：

```
输入: green_mask, highlight_mask, image_shape

1. 连通域分析（connectedComponentsWithStats）
2. 对每个贴边界的连通域：
   a. 计算该连通域内高光 mask 的重叠率
   b. 重叠率 > 阈值（5%）→ 判定为光斑粘连
   c. 膨胀高光 mask（覆盖过渡带）
   d. 从 green_mask 中切除高光重叠区域
3. 形态学开运算断开残留细连接
4. 安全回退：分离后若无合法候选（candidate_filter 返回空）→ 返回原始 mask
```

### 与像素级光照矫正的对比

| 对比维度 | 像素级光照矫正（阶段四） | mask 层面分离（当前方案） |
|----------|---------------------------|---------------------------|
| 作用对象 | V 通道像素值 | 二值 mask 连通域 |
| 对 A 类样本的影响 | 必受影响（全局参数） | 自动跳过（贴边检查不触发） |
| 副作用 | 颜色信号退化、精度下降 | 仅改变连通域拓扑 |
| 回退机制 | 无（像素已修改） | 安全回退到原始 mask |
| 可解释性 | "V 值变了" | "切断了与光斑的连接" |

## 阶段八：实验输出设计

### 三类对比结果

1. **基础算法在 A/B 两类样本上的运行结果**
   - 说明强光斑邻接会导致常规流程失效
   - 验证基础算法的性能边界

2. **算法 A 与算法 B 在 B 类样本上的对比**
   - 说明区域分离算法能否恢复外框检测、孔检测和尺寸测量
   - 这是实验的主结论来源

3. **算法 A 与算法 B 在 A 类样本上的辅助对比**
   - 说明改进算法是否会破坏正常样本
   - 验证精度不退化（不作为主结论，但必须满足）

### 统计指标

| 指标 | 说明 |
|------|------|
| 板检测成功率 | 外框定位成功数/总样本数 |
| 孔检测成功率 | 四孔全部检出数/板检测成功数 |
| X/Y 孔距误差 | 测量值与标称值 94mm 的绝对偏差 |
| 平均绝对误差 | 全部有效测量的误差均值 |
| 重复性标准差 | 多图测量的误差离散度 |
| 强光斑分离是否生效 | B 类样本 region_separation 触发情况 |
| 分离前/后连通域数量 | 定量衡量分离效果 |
| 候选区域面积、宽高比、位置评分 | 分离后候选质量评估 |

### CSV 字段

```
image_name, sample_type, method, board_detect_success, hole_detect_success,
num_holes, pitch_x_mm, pitch_y_mm, abs_error_x_mm, abs_error_y_mm,
mean_abs_error_mm, remark
```
